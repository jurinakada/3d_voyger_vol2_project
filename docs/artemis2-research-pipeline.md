# Artemis II Trajectory Research Pipeline

## Goal

This branch keeps the Unity prototype intact while adding a small research pipeline for
Artemis II / Orion trajectory work. The real rebuild should start from the open
data context in `docs/open-data-context.md`, not from the current Unity CSV.

Target outcome:

- show Orion moving from Earth orbit to lunar flyby and Earth return in Unity / VR
- keep a clear distinction between public NASA trajectory data and simplified Python simulation
- use differences between the public trajectory and the simplified model as educational analysis,
  not as a claim that internal Orion guidance or attitude control has been fully recovered

## Data Levels

1. Public trajectory baseline
   - Source: NASA Artemis II AROW OEM files.
   - Format: CCSDS OEM text with UTC timestamp, Earth-centered EME2000 position, and velocity.
   - Units: km and km/s.

2. Simplified simulation
   - Source: local Python numerical integration.
   - Current file: `scripts/orbit_sim.py`.
   - Physics level: Earth gravity, Moon gravity, initial velocity, numerical integration.

3. Unity visualization CSV
   - Proposed common format:

```csv
time_sec,orion_x_km,orion_y_km,orion_z_km,moon_x_km,moon_y_km,moon_z_km,phase,event
0,7000,0,0,384400,0,0,EarthOrbit,None
```

## Current Repository Gap

The repository currently has a working prototype path:

- `scripts/orbit_sim.py` generates simple CSV files.
- `venture2_project/Assets/Scripts/TrajectoryLoader.cs` reads `time,x,y,z`.
- `venture2_project/Assets/CSV/artemis1_trajectory.csv` is already assigned in Unity.

Important current-data note:

- `output/artemis1_trajectory.csv`
  - generated by `scripts/orbit_sim.py`
  - columns: `time,x,y,z`
  - units: SI meters
  - rows: 36,001
  - time range: 0 to 2,160,000 seconds, or 25 days
  - Earth distance range: about 7,000 km to 503,851 km
- `output/moon_trajectory.csv`
  - generated by `scripts/orbit_sim.py`
  - columns: `time,x,y,z`
  - units: SI meters
  - rows: 36,001
  - Moon distance stays near 384,400 km because the Moon model is circular.
- `venture2_project/Assets/CSV/artemis1_trajectory.csv`
  - columns: `time,x,y,z`
  - units appear to be SI meters, matching the current Unity loader scale.
  - rows: 50,000
  - time range: 0 to 49,999
  - Earth distance range: about 7,001 km to 5,539,535 km
  - this file does not match the current `scripts/orbit_sim.py` output and should not be treated as the canonical Python result until its source is identified.

The research path still needs:

- NASA OEM parser and conversion into Unity-friendly CSV.
- Moon ephemeris aligned to the Orion timestamps.
- phase and event labeling.
- comparison between NASA trajectory and simplified simulation.
- Unity player that uses time-based playback instead of one CSV row per frame.

## Minimal Next Steps

1. Convert a NASA OEM `.asc` file into a normalized Orion CSV.
2. Add Moon positions for the same timestamps using SPICE, Horizons, or a temporary circular approximation.
3. Import the normalized CSV in Unity without changing the current scene behavior.
4. Add a second line renderer for the simplified simulation.
5. Calculate and visualize position differences.
